# Security Policy

## Supported Versions

Security fixes are provided on a best-effort basis for the latest commit on
`main`.

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

Use one of the following private channels:

1. GitHub Security Advisories (preferred): "Report a vulnerability" in the
   repository Security tab.
2. Direct maintainer contact if GitHub Security Advisories are unavailable.

## What to Include

- A clear description of the issue
- Steps to reproduce
- Potential impact
- Suggested remediation (if available)

## Response Expectations

- Initial acknowledgement: best effort, typically within 7 days
- Remediation timeline: depends on severity and maintainer availability
